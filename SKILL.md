# 土拨鼠配图师（Marmot Illustrator）

把一篇文章读透，然后像土拨鼠打洞一样——从地表往下掘进，一层一层挖出文章真正想说的东西，再用温暖手绘线条和固定圆胖土拨鼠画出来。

核心信念：**好配图不是装饰，是一次掘进。** 从表层文字开始，穿过显性信息，到达作者没明说但全文都在呼吸的那个判断。

## 与橙猫版的本质区别

| 维度 | 橙猫版 | 土拨鼠版 |
|---|---|---|
| 风格关键词 | 极简、强线条、冷峻克制 | 温暖、手绘感、环境叙事 |
| 配色方案 | 黑墨线 + 暖橙点睛 | 大地色系（棕/绿/米黄） |
| 构思隐喻 | 建筑师蓝图——精确测量、结构优先 | 掘进者视角——层层深入、发现隐藏 |
| 角色气质 | 温和观察者、略憨拙 | 热情向导、主动参与、带点憨厚幽默 |
| 独有机制 | 无 | 环境层 + 洞口视角 + 掘进深度标记 |
| 适合文章类型 | 技术/产品/分析类偏多 | 生活/成长/叙事/观点类尤其出彩 |

两者不是替代关系。技术架构图用橙猫更干净；需要温度、故事感和情绪共鸣的文章用土拨鼠更有力。

## 总流程：

```
读文章 → 建立文章地图 → 识别图型 → 标记掘进深度 → 选插图位 → 完成构思卡（含环境层）→ 验证可读性 → 写提示词 → 生成图片 → 交付
```

配图构思是内部强制关卡。没有完成构思卡和环境层设计，不得直接写提示词或生成图片。构思通过后默认直接生成，不需要等待用户确认。只有用户明确要求先看方案、只做构思或生成前确认时，才暂停并展示构思卡。

所有文章正文配图统一使用 16:9 横向画幅。图型、内容和构图不得自行改用 3:2、方形或竖图。只有用户明确指定其他比例，或任务明确是公众号封面时才例外。

## 第一步：读文章，建立文章地图

通读全文，至少两遍。第一遍看它在讲什么，第二遍找下面这些东西：

1. **文章主命题**：全文最终希望读者相信什么，只能写一句。
2. **段落子命题**：每个重要段落推进了哪一步。
3. **显性隐喻**：作者自己写出来的比喻和意象。
4. **隐性隐喻**：作者没明说，但全文都围绕的感觉。
5. **情绪峰值**：转折、袒露脆弱、具体细节出现或结尾收束的位置。
6. **原文指纹**：反复出现或高度独特的物件、动作、案例和领域特征。

先产出一份文章地图：

| 段落位置 | 这一段在做什么 | 单句子命题 | 原文指纹 | 信息结构 | 情绪变化 | 掘进深度 |
|---|---|---|---|---|---|---|
| 引用锚点句 | 提出问题、解释机制、完成转折或收束 | 只写一个判断 | 具体物件、动作、案例或领域特征 | 顺序、层级、对比、连接、组成或隐喻 | 从什么变成什么 | 地表 / 浅层 / 中层 / 深层 |

**掘进深度**是土拨鼠版的独有维度，表示这段内容在文章中的认知层次：

| 深度 | 含义 | 视觉暗示 |
|---|---|---|
| **地表** | 文章明确说出来的东西 | 明亮、开阔、可见 |
| **浅层** | 稍微往下挖一步就能看到的联系 | 草根、表层土壤、光线透过 |
| **中层** | 需要一些推理才能到达的判断 | 泥土层、有质感但不黑暗 |
| **深层** | 文章最核心、作者可能自己都没完全意识到的信念 | 更深土壤、但永远有光从上面照下来 |

**原文指纹**是读者看见后能联想到文章具体内容的线索，可以是：

1. 文中明确出现的物件或动作。
2. 文章独有的关系。
3. 文章反复讨论的领域特征。

优先选择具体、独特、可画的指纹。不要只留下抽象词。

**注意**：不要提取那些只是修辞漂亮但没有情绪重量的句子。判断标准是：删掉这个意象，文章的感觉会不会变淡。会，就留；不会，就放过。

## 第二步：识别图型

文章插图不只有隐喻画面。先识别段落的信息结构，再选择最合适的图型：

| 图型 | 识别信号 | 主要回答 |
|---|---|---|
| **流程图** | 先、再、然后、最后、循环、验证、步骤 | 事情按什么顺序发生 |
| **架构图** | 系统、模块、层、输入输出、数据流、服务 | 各部分如何协作 |
| **对比图** | 过去与现在、A 与 B、不是而是、优缺点 | 两者到底哪里不同 |
| **关系图** | 影响、依赖、映射、因果、网络、连接 | 对象之间如何关联 |
| **结构图** | 组成、内部、分层、包含、拆解、剖面 | 一个整体由什么构成 |
| **概念插图** | 情绪、隐喻、转折、案例、余味 | 这段话让人理解或感受什么 |

图型判断优先于风格判断。

同一段可能同时含有多种结构。选择最能帮助读者理解作者核心判断的主图型，必要时加入一种辅助结构，但不能把所有图型混在一张图里。

详细识别与规范见 `references/diagram-types.md`。

所有图型统一使用一种**温暖手绘解释图**语言：保留信息的准确性，同时注入温度和场景感。不追求冰冷的精确，但也不允许模糊。

## 第三步：选择配图任务

不是先找漂亮句子，而是先判断哪里需要图片帮助读者理解。每个候选位置必须属于以下一种任务：

1. **解释**：把抽象机制变成可见关系。
2. **区分**：让读者看见两个概念到底哪里不同。
3. **转折**：标出文章从旧判断进入新判断。
4. **记忆**：把关键案例变成读者记得住的画面。
5. **收束**：给结尾留下情绪余味。

一张图只能承担一个主要任务。

从候选位置中选 2 至 4 个。短文选 1 至 2 个，长文选 3 至 4 个，宁缺毋滥。

两张插图之间至少隔开三四段正文。

每个位置标注清楚：插在哪一段之后（引用该段最后一句作为锚点），以及该位置的**掘进深度**。

## 第四步：完成配图构思（含环境层设计）

这是整个 Skill 最关键的一步。每张图都要先完成一张构思卡。

### 4.1 压缩成单一命题

先完成这句话：

```
这张图不是要画______，而是要让读者感受到______。
```

注意：橙猫版用的是"看懂"，土拨鼠版用"感受到"。一字之差，区别在于：
- "看懂"偏重**信息传递**
- "感受到"偏重**理解 + 共鸣**

### 4.2 标记掘进深度

根据第一步文章地图中的掘进深度，确定这张图的视觉层次：

| 掘进深度 | 光线处理 | 场景元素 | 色温 |
|---|---|---|---|
| 地表 | 明亮均匀 | 天空、草地、远山 | 暖白/淡黄 |
| 浅层 | 有方向性的柔和侧光 | 草根、薄土、小石子 | 米黄/浅棕 |
| 中层 | 定点光源从上方投射 | 泥土纹理、根系、小洞穴入口 | 棕色/橄榄绿 |
| 深层 | 一束光从上方窄缝照入 | 深土、但总有光的希望 | 深棕/暖黄光晕 |

掘进深度影响整张图的氛围基调，但**不能喧宾夺主**。环境是背景衬布，不是主体。

### 4.3 按图型选择表达单位

不同图型使用不同表达单位：

1. **流程图**：状态节点、方向连接、起点和终点。
2. **架构图**：模块边界、层级、接口和数据流。
3. **对比图**：同一比较维度下的左右两侧。
4. **关系图**：核心对象、关系边和方向或强弱。
5. **结构图**：整体轮廓、组成部分和包含关系。
6. **概念插图**：主语、对照物、变化线和见证者。

### 4.4 设计环境层（土拨鼠版独有）

环境层是土拨鼠版区别于橙猫版的核心特征。每张图必须设计一个**轻量环境暗示**，它不做主角，但提供情绪基调和空间锚定：

#### 环境层设计原则

1. **环境元素不超过画面面积的 20%**。它是底色，不是内容。
2. **环境元素必须是具体的、可画的**。"温暖的氛围"不行，"右下角三根草叶"可以。
3. **环境和掘进深度一致**。地表深度不放泥土元素，深层不放蓝天白云。
4. **环境元素不能和原文指纹竞争注意力**。如果原文指纹是"液态玻璃"，环境就不能也是玻璃质感的物。

#### 环境元素库

| 掘进深度 | 可用元素 | 示例 |
|---|---|---|
| 地表 | 天空一角、远处草坡、几朵云、阳光射线 | 右上角露出一点淡蓝天空和一朵胖云 |
| 浅层 | 草叶边缘、散落的小石子、露珠、草根 | 底部边缘画出三四根草叶，两颗小石子 |
| 中层 | 泥土纹理、蚯蚓痕迹、小植物根部、洞穴边缘 | 左下角一小块有质感的泥土断面 |
| 深层 | 深色土壤、一根从上面垂下来的根、远处的微光 | 四周压暗，顶部中央一束暖光射入 |

#### 洞口视角（可选高级技巧）

当文章涉及"表象 vs 本质"、"说出的 vs 没说的"、"公开的 vs 私密的"这类二元对立时，可以使用**洞口视角**：

- 画面上半部：明亮的地表世界（代表显性内容）
- 画面下半部：稍暗的洞穴内部（代表隐性含义）
- 土拨鼠站在洞口位置，同时能看到两边

这不等于简单的上下分割。洞口是一个**过渡地带**，有光线渐变、有泥土的弧形边缘、有从上面照下来的光柱。

### 4.5 控制重复元素

1. 同类大型元素默认不超过 3 个。
2. 同类小型元素超过 5 个时，改用数量变化、集合轮廓、疏密、尺寸递减或单个代表元素。
3. 同一流程的节点可以保持统一外框，但内部图形、留白和轮廓节奏要有明确差异。
4. 对比图两侧需要对齐，但不要求复制完全相同的对象。
5. **土拨鼠只出现一次**，不为每个步骤复制一只土拨鼠。

构思完成后做**重复审美测试**：眯眼看画面，如果首先看到的是一排相同盒子而不是核心关系，必须重新抽象。

### 4.6 控制视觉重量

1. 单个普通模块默认不超过画面面积的 15%。
2. 核心模块可以达到约 20%，但不能同时又拥有最粗线条、最多细节和最强颜色。
3. **大地色系的棕色只强调关键节点**，不通过大面积填色制造重量。

做**眯眼平衡测试**：眯眼看时，是否有一个元素变成无法忽略的巨大色块？如果有，缩小它、减少细节或降低线条重量。

### 4.7 概念插图的视觉角色

每张图最多使用四类视觉角色，不要求全部出现：

1. **主语**：读者第一眼应该看见什么。
2. **对照物**：什么东西帮助主语表达差异。
3. **变化线**：箭头、路径、分叉、距离、大小、遮挡或前后关系。
4. **见证者**：圆胖土拨鼠承担热情向导、主动参与或尺度参照。

主语只能有一个。对照物最多一个。

优先把命题变成一个可以看懂的动作：掀开、穿过、筛掉、连接、抬起、埋下、对准、**咬断**（土拨鼠独有——用大板牙咬穿障碍物）、**刨开**（土拨鼠独有——用爪子刨开覆盖物）。

### 4.8 选择关系语法

画面是一句视觉句子。根据核心命题只选择一种主要关系语法：

1. **前后变化**：从旧状态到新状态。
2. **表层与深层**（土拨鼠版特别擅长）：表面行为下面藏着真实原因——天然契合掘进隐喻。
3. **分叉与选择**：同一输入通向多个假设。
4. **发散与收敛**：先提出可能，再筛到少数结果。
5. **阻隔与穿越**：被某种限制挡住，或找到新的通道。
6. **尺度与距离**：领先、追赶、压迫、微小或开阔。
7. **积累与生长**：小动作经过时间形成结果。
8. **循环与验证**：提出、回查、修正、再输出。

详细方法见 `references/conception-protocol.md`。

### 4.9 填写构思卡

每张图必须输出：

| 字段 | 内容要求 |
|---|---|
| 插入位置 | 引用前一段最后一句 |
| 图型 | 流程图、架构图、对比图、关系图、结构图或概念插图 |
| 画幅 | 固定 16:9 横向，除非用户明确指定其他比例 |
| 配图任务 | 解释、区分、转折、记忆或收束，只选一个 |
| 单一命题 | 读者看完图应该感受到的一句话 |
| 原文指纹 | 至少一个只能来自这篇文章的线索 |
| 掘进深度 | 地表 / 浅层 / 中层 / 深层 |
| 信息单位 | 节点、模块、比较维度、关系对象、组成部分或视觉主语 |
| 组织规则 | 顺序、层级、左右对照、网络连接、包含关系或关系语法 |
| 环境层设计 | 具体的环境元素、位置和面积占比（土拨鼠版必填） |
| 重复元素预算 | 哪些元素会重复、最多几个、超过后如何抽象 |
| 视觉重量 | 各主要结构的大致面积、线条和颜色权重 |
| 手写标签 | 是否需要，最多几个，每个最多多少字 |
| 标签规划 | 每个标签的文字、作用、颜色、位置和能否删除 |
| 阅读顺序 | 第一眼、第二眼、最后理解什么 |
| 土拨鼠动作 | 观察、操作或见证什么，**必须具体且有趣** |
| 一秒读图目标 | 一秒能看出的差异或变化 |
| 三秒读图目标 | 三秒能感受到的文章判断 |
| 删除项 | 为了保持单一命题，主动不画什么 |
| 画面一句话 | 用一句话描述最终画面 |

### 4.10 构思验证

构思必须同时通过八个测试：

1. **一秒测试**：缩小看，能否立刻分辨主语和主要变化？
2. **三秒测试**：不看说明，能否说出接近原文的感受？
3. **换文测试**：把图放进另一篇同主题文章，是否仍然完全成立？如果成立，原文指纹太弱。
4. **删减测试**：删掉任意一个次要元素，意思是否不变？如果不变，就应该删掉。
5. **图型测试**：这个段落是否能被另一种图型表达得更直接？如果能，重新选择图型。
6. **无文测试**：不依赖长文字标签时，结构是否仍然清楚？
7. **环境层测试**（土拨鼠版独有）：去掉环境层后，画面是否仍然成立？环境是否在抢戏？如果环境比内容更吸睛，减淡或简化环境元素。
8. **眯眼平衡测试**：是否有单个元素因过大、过粗、过密或颜色过强而压住其他信息？

任一测试不通过，回到单一命题重新构思。

### 4.11 进入生成

构思卡和八项测试通过后直接进入提示词与图片生成，不等待用户确认。用户明确要求先看方案时再展示构思卡并暂停。

### 固定角色规则

默认主视觉角色是 `assets/marmot-main-visual.jpg` 中的**圆胖土拨鼠**。生成前必须读取这张参考图，让图片模型把它作为角色外形参考。

圆胖土拨鼠的固定特征：

1. **圆滚滚、矮胖的身材**，肚子明显，四肢短小可爱。
2. **棕色皮毛为主色调**，背部和头顶为较深的棕褐色，胸腹为浅米黄色。
3. **标志性的大门牙**（两只长方形上门齿），笑的时候会露出来。
4. **绿色针织毛线帽**（带一个小绒球），这是标志性配饰。
5. **又大又圆的黑眼睛**，眼神友善、充满好奇，偶尔有点憨憨的感觉。
6. **短小的圆形耳朵**藏在毛发中若隐若现。
7. **小巧的黑鼻子**和几根胡须。
8. **表情丰富自然**——可以微笑、惊讶、认真、好奇，但整体保持温暖友好的气质。
9. **短尾巴**，小圆脚掌。

角色使用规则：

1. 文章需要动作、陪伴、尺度或方向时，优先由圆胖土拨鼠承担，不再默认生成人类。
2. **土拨鼠的动作要充分利用其物种特征**：
   - **刨开**：用前爪扒开表面的覆盖物（适合理清表象 vs 本质）
   - **咬断**：用大门牙啃穿障碍物（适合突破限制）
   - **探头**：从洞口探出头张望（适合发现新事物）
   - **招手**：举起一只前爪打招呼（适合引荐、介绍）
   - **抱住**：双臂环抱某个物体（适合珍惜、保护）
   - **蹲坐**：圆滚滚地坐着观察（适合沉思、见证）
3. 根据原文改变土拨鼠的姿势和表情，但不能改变固定体型、配色、大门牙和绿色帽子特征。
4. 角色默认占画面约 8% 至 12%（比橙猫稍大一点点，因为土拨鼠本身更有存在感）。除非文章核心就是角色本身，否则不要让土拨鼠压过原文意象。
5. 允许给土拨鼠添加**一个小道具**，而且比橙猫版宽松——因为土拨鼠的性格更适合"带着工具干活"：小铲子、放大镜、小旗子、背包、笔记本。
6. 不把土拨鼠画成写实啮齿动物、水豚、仓鼠或普通卡通老鼠。
7. 用户明确要求人物或其他主体时，遵循用户要求，不强行使用土拨鼠。

详细的视觉转译技巧和正反例，看 `references/visual-translation.md`。

## 第五步：把构思翻译成提示词并生成图片

风格基线（所有图共用）：**温暖手绘风格、有质感的墨线、柔软的线条变化、大地色系配色、带有轻微的环境氛围**。画面依靠清晰轮廓和温暖调性成立，不依赖冷峻的极简或复杂的水彩铺色。完整的风格定义和提示词模板在 `references/prompt-templates.md`，**生成前必读**，提示词按模板组装，不要凭感觉现写。

提示词必须严格来自已经确认的构思卡。不要在写英文提示词时临时添加新物件、新隐喻或第二个观点。

文章解释图允许少量手写风格中文短标签。标签用于帮助读者快速定位结构：

1. 每张图默认 2 至 5 个标签。
2. 每个标签优先 2 至 4 个汉字，最多 6 个汉字。
3. 使用自然、清楚、略有手写感的中文。
4. 标签贴近对应结构。

标签语义采用**大地色系**的颜色分工：

1. **深棕色**：对象名称和普通状态（替代黑色，更温暖）。
2. **绿色**（帽子同色）：主路径、关键动作和希望读者记住的节点。
3. **暖橙色**：问题、阻塞、淘汰或风险（像夕阳一样醒目但不刺眼）。
4. **淡蓝色**：补充状态、外部条件或次要解释（像天空）。

一张图通常只使用深棕色加一种功能色。

图片生成的调用方式，按这个优先级：

1. **用户环境里如果有图片生成的 MCP 工具或其他图片 Skill**（如 ImageGen、Flux 等），直接调用，提示词用英文版模板。
2. **如果没有任何图片生成工具**：不要假装生成。把每张图的完整提示词（中英双语）整理好交给用户，告诉用户可以贴到各种图片生成工具里使用。

所有文章正文配图固定使用 16:9 横向画幅。推荐尺寸为 `1536 × 864` 或其他严格 16:9 尺寸。

生成后自查：

1. 画面里有没有意外出现文字或乱码笔画？
2. 主体是否清晰，环境层是否提供了正确的氛围而不抢戏？
3. 原文指纹是否真的生成出来？
4. 不看说明时，画面是否仍能让熟悉文章的人联想到对应段落？
5. 画面是否同时包含内容关联、情绪共鸣和环境氛围？
6. 如果使用土拨鼠，**体型、棕色皮毛、大门牙、绿色毛线帽和大圆眼睛**是否与参考图一致？
7. 线条是否有温暖的手绘感，而不是冷峻的几何线条？
8. 大地色系是否正确使用？有没有出现不该有的冷色调？
9. 如果包含节点、模块或关系线，是否符合 `references/diagram-types.md` 的 QA？
10. 输出尺寸是否为严格 16:9？
11. 是否出现过多几乎相同的元素？
12. 环境层的元素是否和掘进深度匹配？

第 3 或第 4 项不合格时，把原文指纹前移、写得更具体后重出。最多重试两次，仍不理想就如实告诉用户并给出手动调整建议。

## 第六步：交付

交付物一律打包清楚：

1. **图片文件**（如果成功生成），命名 `01-插图位置关键词.png` 格式。
2. **构思卡存档**：保留每张图的图型、单一命题、原文指纹、组织规则、掘进深度和环境层设计。
3. **插图说明清单**：每张图对应插在哪段之后、画面是什么、为什么放这里。
4. **提示词存档**：每张图的完整提示词。

## 边界

- 文章如果是纯信息向的（产品参数评测、操作教程），如实告诉用户这类文章更适合橙猫版的极简线条风格，问清楚是否仍要继续。
- 不画真实公众人物的脸。文中提到具体人物时，用背影、剪影或象征物代替。
- 不在画面中复刻受版权保护的角色和 IP。
- **不画恐怖、恶心或阴暗的画面**。即使文章讨论沉重话题，土拨鼠版也坚持"深层总有光"的原则——最深的洞也一定有一束光从上面照下来。
