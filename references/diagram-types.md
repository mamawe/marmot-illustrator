# 常用图型识别与绘制规范（土拨鼠版）

文章地图完成后，先判断段落的信息结构。图型不是装饰风格，而是信息组织方式。

所有正文图型统一使用 16:9 横向画幅。

所有图型统一使用**温暖手绘解释图**语言：结构准确，但注入温度和场景感。

## 一、图型路由

按以下顺序判断：

1. 是否存在明确时间或操作顺序？ → 流程图
2. 是否在解释一个系统由多个模块协作？ → 架构图
3. 是否围绕同一维度比较两个方案？ → 对比图
4. 是否强调多个对象之间的影响、依赖或映射？ → 关系图
5. 是否在拆解一个整体的内部组成或层级？ → 结构图
6. 以上都不是 → 概念插图

## 二到七、各图型规范

（结构型图型的必备字段、构图规则与橙猫版基本一致，详见原版 diagram-types.md。以下仅列出土拨鼠版的差异和补充。）

### 二、流程图

**土拨鼠差异**：
- 橙猫用法：位于关键动作节点。
- **土拨鼠用法**：可以在流程的"掘进"环节出现——比如在"分析"或"验证"阶段用刨或探头的动作。也可以作为流程的"起点见证者"坐在第一个节点旁边。
- 节点形状可以略微圆润（配合温暖风格），但仍需保持一致。

### 三、架构图

**土拨鼠差异**：
- 可以站在系统边界的外面（像是在观察一个蚁穴），或者从一个角落的"洞口"探出头来观察。
- 系统边界可以用**泥土断面式的弧形边缘**来表达层次感，而非简单的矩形框。

### 四、对比图

**土拨鼠差异**：
- 可以站在中央分割线上（像裁判，但更友善）。
- 或者在其中一侧执行"刨开"动作揭示差异。
- 中央分割线可以用一条**地面裂缝**或**草地的自然分界**来表达，而非生硬的直线。

### 五、关系图

**土拨鼠差异**：
- 可以用目光连接两个对象（土拨鼠转头看两边）。
- 或者用一只前爪搭在一个关系线上（"我确认这条连接存在"）。
- 关系线的线条可以有轻微的**有机弯曲**，不必是笔直的。

### 六、结构图

**土拨鼠差异**：
- **完美契合掘进隐喻**——剖面图可以直接画成"土拨鼠打洞挖出来的断面"。
- 外壳可以被土拨鼠掀开（像刨开草皮一样）。
- 内部组成可以用"不同深度的土层"来区分层次。

### 七、概念插图

遵循 `conception-protocol.md` 的掘进式构思方法。土拨鼠版的概念插图是最强的差异化领域——当不需要严格的信息结构时，温度和环境层可以最大化发挥。

## 八到十三、通用规范

重复元素预算、视觉重量、手写标签、避免 PPT 感等规范与橙猫版一致，以下补充土拨鼠版特有的 QA 条目：

### 结构表达 QA（补充）

1~15 条与橙猫版一致，以下是新增：

16. **土拨鼠是否参与了有意义的动作？**（不只是站着）
17. **环境层元素是否和掘进深度匹配？**
18. **大地色系是否正确使用？有没有出现不该有的冷色调？**
19. **线条是否有温暖的手绘感？**
20. **整体画面是否传递了正确的情绪温度？**
